Vite 介绍

Vite 是一款前端构建工具 ，由 Vue 作者尤雨溪开发，旨在提供快速的开发体验，核心特点是 “快速的开发服务器启动” 和 “高效的热模块替换（HMR）”。不同的阶段调用不同的钩子函数。

核心特性

  1. 极快的开发服务器启动时间：
    • 开发时不打包代码，而是直接利用浏览器原生的 ES 模块（import/export）能力，通过 HTTP 服务器提供文件，实现毫秒级启动。
  2. 即时热模块替换（HMR）：
    • Vite 提供了几乎即时的热模块替换功能，当你修改代码时，仅更新发生变化的模块，保持页面状态，从而显著提高开发效率。
  3. 开发阶段：基于原生 ES Modules
    • Vite 充分利用了现代浏览器对 ES（import/export） 模块的原生支持，避免了传统的打包过程，通过 HTTP 服务器提供文件，实现毫秒级启动，使得开发和生产环境的表现更加一致。
  4. 生产阶段：基于 Rollup 打包
    • 生产环境下使用 Rollup 进行打包，利用其 Tree-shaking 能力移除未使用代码，生成高度优化、体积更小的最终产物。
    • 支持多种输出格式，并针对现代浏览器进行优化（如自动分割代码、生成 ES 模块输出等）。
  5. 开箱即用的功能
    • 内置对 TypeScript、JSX、CSS、图片等资源的处理，无需额外配置。
    • 支持环境变量、别名配置、CSS 预处理器（Less/Sass）等常用需求。
  6. 丰富的插件生态
    • Vite 支持丰富的插件生态系统，拥有专属 Vite 插件体系，兼容 Rollup 插件（需适配），可扩展处理各类场景（如 HTML 注入、PWA 支持等）。

与传统工具（如 Webpack）的核心差异

维度 Vite 传统工具（如 Webpack）
开发启动方式 基于原生 ES module 直接服务文件，不打包 先打包所有模块再启动服务
启动速度 毫秒级（大型项目优势更明显） 随项目规模增长变慢
HMR 性能 精准更新，速度极快 需遍历依赖树，大型项目较慢
生产构建 Vite 在开发环境下使用 esbuild 进行预构建，生产环境下使用 Rollup 进行打包（esbuild 是用 Go 语言编写的，可以充分利用多核 CPU 的优势从而加快构建速度，）输出更精简 Webpack 是基于 Node.js 运行的，自带打包器，输出包含较多运行时代码
生产环境打包 在生产环境下使用 Rollup 进行打包，Rollup 生成的文件通常更小，但 Vite 的插件 API 与使用 esbuild 作为打包器不兼容，因此目前使用 Rollup 打包是其核心功能之一，提供了丰富的优化选项，如代码分割、压缩等。
配置复杂度 Vite 设计为更易于配置和使用，内置了许多常用的功能，减少了配置的复杂性，使得开发者可以更快地开始项目开发 Webpack 提供了高度可配置的接口，可以满足各种复杂的构建需求，但这也意味着配置可能会变得复杂，特别是对于新手。

适用场景

  • 现代前端项目开发（支持 Vue、React、Svelte 等框架）。
  • 对开发效率要求高的团队或项目（快速启动、实时反馈）。
  • 需要优化生产环境构建产物体积的场景。

Vite 与 Vue CLI 的提升和区别

Vue CLI 是 Vue.js 的一个官方脚手架工具，提供了项目初始化、开发服务器启动、构建等功能。但随着项目规模的增大，Vue CLI 在开发服务器启动和热更新方面逐渐显现出性能瓶颈。Vite 的出现正是为了解决这些问题。

性能提升

Vite 相较于 Vue CLI 在性能上有显著提升，主要体现在以下几个方面：

1.开发服务器启动时间：

Vite 利用 ES 模块的特性，避免了传统打包工具（如 Webpack）的打包过程，从而使开发服务器的启动时间大大缩短。一个简单的 Vue 项目，Vite 可以在几百毫秒内启动，而 Vue CLI 可能需要几秒钟。

Vue CLI 在开发模式下使用 Webpack 进行模块打包，这会导致启动时间较长，尤其是在大型项目中。尽管 Vue CLI 提供了热模块替换（HMR），但在处理大量文件时，HMR 的性能可能不如 Vite。

2. 热更新速度

Vite 的热更新速度几乎是实时的，因为它仅重新加载发生变化的模块。而 Vue CLI 在进行热更新时，可能需要重新打包整个应用，导致更新速度较慢。

3.构建速度

在生产环境中，Vue CLI 使用 Webpack 进行构建，这通常需要较长的时间，尤其是在大型项目中。Webpack 的构建过程包括代码分割、压缩、Tree-shaking 等，虽然功能强大，但速度相对较慢。

Vite 在生产环境中使用 Rollup 进行构建，Rollup 以高效和快速著称。Vite 的生产构建速度通常比 Vue CLI 快，尤其是在大型项目中。

4.插件生态

Vue CLI 拥有丰富的插件生态系统，提供了大量的官方和社区插件，可以满足各种开发需求。Vue CLI 的插件系统基于 Webpack，因此可以集成各种 Webpack 插件。

Vite 也有一个不断增长的插件生态系统，支持 Vue、React、Svelte 等多种框架。Vite 的插件系统设计得非常灵活，允许开发者根据需要扩展构建过程。
